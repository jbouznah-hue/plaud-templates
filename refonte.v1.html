Pourquoi TranscriptAI ?

Problème : Plaud coûte $12/mois avec stockage limité et dépendance cloud. tl;dv facture $25+/mois. Les deux ont des transcriptions médiocres en hébreu et une intelligence limitée. Impossible de croiser les données entre les deux outils. 390 fichiers Plaud + 467 réunions tl;dv = archive dispersée impossible à exploiter.
Solution : Un pipeline unifié auto-hébergé sur Hetzner. Groq Whisper pour la transcription (35x moins cher qu'OpenAI, 216x temps réel), pyannote pour la diarisation, Claude pour l'intelligence. Toutes vos réunions dans une seule interface, cherchables, annotées, exportables.

STEP 1 — Validation intake

Vérifications effectuées :

  • Existence fichier : MinIO object exists → 404 → job.status = 'failed', reason = 'file_not_found'
  • Format détection : python-magic (file magic bytes, pas l'extension) → détecte audio/video/inconnu
  • Taille : > 0 bytes, < 2GB (configurable MAX_FILE_SIZE_GB)
  • Durée estimée : ffprobe -v error -show_entries format=duration -of json → duration_seconds stocké
  • Lisibilité : FFmpeg probe test → détecte fichiers corrompus
# Python FastAPI
async def validate_intake(job_id: str) -> ValidationResult:

STEP 3 — Normalisation FFmpeg + extraction

Conversion universelle : n'importe quel format → WAV 16kHz mono 16-bit PCM.

# Commande complète de normalisation
ffmpeg \
  -i "{input}" \
  -vn \                              # Supprimer piste vidéo
  -ac 1 \                            # Mono (1 canal)
  -ar 16000 \                        # 16kHz sample rate (optimal Whisper)
  -acodec pcm_s16le \                # 16-bit PCM little-endian
  -af "loudnorm=I=-16:TP=-1.5:LRA=11:print_format=json" \  # Normalisation -16 LUFS
  "{output.wav}"

# Si VAD pre-filter activé (enlève silences >500ms)
-af "silenceremove=start_periods=1:start_duration=0.5:start_threshold=-40dB,
     loudnorm=I=-16:TP=-1.5:LRA=11"

STEP 5 — Découpage en chunks avec overlap

Stratégie de découpage (F21 — overlap 3s)

# Paramètres configurables
CHUNK_DURATION = 600  # 10 minutes (défaut, range 300-900s)
OVERLAP = 3          # 3 secondes de chevauchement (défaut)

# Calcul des segments
chunks = []
pos = 0
while pos < total_duration:
    start = max(0, pos - OVERLAP)
    end = min(total_duration, pos + CHUNK_DURATION + OVERLAP)
    chunks.append({
        "index": len(chunks),
        "start_s": start,
        "end_s": end,
        "logical_start": pos,      # Pour déduplication
        "logical_end": pos + CHUNK_DURATION
    })
    pos += CHUNK_DURATION

# Exemple pour 35 minutes:
#  chunk_0: 0:00 - 10:03  (logical: 0-10)
#  chunk_1: 9:57 - 20:03  (logical: 10-20)
#  chunk_2: 19:57 - 30:03 (logical: 20-30)
#  chunk_3: 29:57 - 35:00 (logical: 30-35)

Déduplication overlap :

def merge_chunks(chunks_results: list) -> list:
    all_words = []
    for chunk in sorted(chunks_results, key=lambda x: x["chunk"]["logical_start"]):
        logical_start = chunk["chunk"]["logical_start"]
        logical_end = chunk["chunk"]["logical_end"]
        for word in chunk["result"]["words"]:
            abs_start = word["start"] + chunk["chunk"]["start_s"]
            abs_end = word["end"] + chunk["chunk"]["start_s"]
            # Garder seulement les mots dans la zone logique (pas l'overlap)
            if logical_start <= abs_start < logical_end:
                all_words.append({**word, "abs_start": abs_start, "abs_end": abs_end})
    return all_words  # Transcript continu sans doublons

  2. Segmentation — réseau de neurones pyannote : découpe en segments monospeaker
  3. Embedding extraction — ResNet256 : vecteur 192 dimensions par segment

Logique d'attribution speaker :

Pour chaque mot [start, end], on cherche le segment pyannote qui couvre la plus grande partie de ce mot. Si plusieurs speakers couvrent le même mot (turn change), on prend celui avec la plus grande intersection temporelle.

UC01 — Import batch archive Plaud (390 fichiers)

Déclencheur

Jeremy veut archiver ses 390 enregistrements Plaud des 2 dernières années

Temps total

~6-8 heures de traitement (non-bloquant, en arrière-plan)

Coût estimé

~$12-18 Groq + ~$2-5 Claude (notes auto si workflow actif)

Étapes :

  1. Jeremy va dans Batch Upload → sélectionne les 390 fichiers .ogg depuis son Mac
  2. L'UI affiche une progress bar globale (0/390) + ETA estimé
  3. FastAPI crée 390 jobs Celery en queue avec priorité normale
  4. Celery workers (concurrency=2) traitent 2 fichiers simultanément
  5. Chaque fichier : validation → FFmpeg → Groq (1-2 min/h audio) → pyannote → alignment → Claude
  6. Dashboard monitoring : chaque job visible avec statut, durée, erreurs éventuelles
  7. Les fichiers failés (corrompus, trop courts) sont listés séparément pour review
  8. Après completion : 390 réunions dans la bibliothèque, cherchables par texte
Résultat : 2 ans d'enregistrements Plaud archivés, cherchables, avec speakers identifiés pour les habitués. Rechercher "Merkasa funding" retrouve toutes les réunions pertinentes.

FFmpeg & Formats audio/vidéo

Qu'est-ce que FFmpeg ?
FFmpeg est le couteau suisse universel de l'audio et de la vidéo. C'est un logiciel open source (100% gratuit) qui comprend et convertit TOUS les formats audio/vidéo qui existent. Quand votre fichier arrive dans TranscriptAI, FFmpeg le lit, extrait l'audio, et le convertit en WAV 16kHz mono — le format optimal pour Whisper. L'utilisateur n'a jamais à se soucier du format de son fichier.

Cas particuliers importants

Vocaux WhatsApp

Format : .ogg avec codec Opus. FFmpeg les décode nativement. Commande : ffmpeg -i voice.ogg -ac 1 -ar 16000 -acodec pcm_s16le out.wav

Enregistrements Plaud

Format : .ogg avec codec Vorbis. Qualité 32-64kbps. Support natif FFmpeg sans librairie additionnelle.

Exports tl;dv

Format : .mp4 H.264. FFmpeg extrait la piste audio avec -vn (no video), conversion immédiate.

Enregistrements Zoom

Format : .mp4 ou .m4a. Parfois multi-pistes (speaker séparé). FFmpeg fusionne automatiquement avec -ac 1.

Chrome Extension (bot)

Capture via Web Audio API → .webm (codec Opus ou Vorbis selon navigateur). FFmpeg supporte les deux.

iPhone Voice Memos

Format : .m4a (AAC). Excellent pour la transcription, qualité micro iPhone souvent meilleure que Plaud.

Commandes FFmpeg de référence

Conversion principale (pipeline TranscriptAI) 
# Conversion universelle → WAV optimal Whisper
ffmpeg -i "{input}" \
  -vn \                              # Pas de vidéo
  -ac 1 \                            # Mono
  -ar 16000 \                        # 16kHz
  -acodec pcm_s16le \                # 16-bit PCM little-endian
  -af "loudnorm=I=-16:TP=-1.5:LRA=11" \  # Normalisation LUFS
  "{output.wav}"

# Taille résultante: 10min = 19.2MB | 1h = ~115MB
Analyse qualité audio 
# Loudness analysis (LUFS)
ffmpeg -i input.wav -af loudnorm=print_format=json -f null - 2>&1 | tail -12

# Silence detection
ffmpeg -i input.wav -af silencedetect=noise=-40dB:duration=0.5 -f null - 2>&1

# Duration + format info
ffprobe -v quiet -print_format json -show_format -show_streams input.wav

# Peak level
ffmpeg -i input.wav -af astats=metadata=1:reset=1 -f null - 2>&1 | grep Peak_level
Extraction WhatsApp vocal 
# WhatsApp .ogg (Opus codec) → WAV Whisper
ffmpeg -i whatsapp_voice.ogg \
  -ac 1 -ar 16000 -acodec pcm_s16le \
  output.wav

# Vérifier le codec avant traitement
ffprobe -v error -select_streams a:0 \
  -show_entries stream=codec_name,codec_long_name \
  -of default=noprint_wrappers=1 whatsapp_voice.ogg
# Sortie attendue: codec_name=opus

Questions techniques résolues

QuestionRéponse
Pourquoi chunk 10min et pas 25min (limite Groq) ?WAV 16kHz 10min = 19.2 MB → sous la limite 25 MB Groq. 15min = 28.8 MB → dépasserait. De plus, chunks plus courts = retry plus rapide et parallélisme plus efficace.
Overlap 3s — ne pas envoyer l'overlap à Groq pour économiser ?L'overlap est DANS le chunk envoyé à Groq (on envoie les 3s extra). La déduplication retire les mots en double APRÈS la transcription. C'est intentionnel : on préfère payer 6s de transcription en plus plutôt que perdre des mots aux bordures.
Temps de démarrage à froid mode offline (faster-whisper) ?faster-whisper large-v2 : ~15-20s de chargement du modèle sur CPU. Medium : ~8s. Pour usage fréquent, conserver le modèle en RAM est recommandé (coût : 1-3 GB RAM permanent).
Peut-on lancer pyannote et Demucs simultanément ?Techniquement oui, mais 2×(2-4 GB) = 4-8 GB → sature les 8 GB disponibles. CONFIGURATION RECOMMANDÉE : exécuter Demucs AVANT pyannote séquentiellement dans le même job. Le gain de temps n'en vaut pas le risque OOM.
pgvector indexation performance pour 10 000 speakers ?Index IVFFlat avec 100 listes : ~10ms par query. Reconstruire l'index tous les 1000 nouveaux speakers avec VACUUM ANALYZE.
Limite Groq API rate limit ?Free tier : 100 req/min, 2000 req/jour. Paid : beaucoup plus élevé. Pour batch 390 fichiers (4 chunks/fichier = 1560 requêtes) : ~16 min sur free tier. Upgrade vers paid pour batch massif.

Schéma PostgreSQL

CREATE TABLE user_quotas (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id           UUID UNIQUE REFERENCES users(id) ON DELETE CASCADE,
  
  -- Allocation
  hours_allocated   FLOAT NULL,          -- NULL = illimité (super admin Jeremy)
  
  -- Consommation
  hours_used_month  FLOAT DEFAULT 0.0,   -- Cumulé depuis le dernier reset
  hours_used_total  FLOAT DEFAULT 0.0,   -- Toutes périodes confondues
  
  -- Reset mensuel
  reset_day         INT DEFAULT 1,       -- 1er du mois (configurable 1-28)
  last_reset_at     TIMESTAMPTZ DEFAULT NOW(),
  
  -- Politique dépassement
  overage_policy    TEXT DEFAULT 'warn', -- 'block' | 'warn' | 'allow'
  
  -- Alertes
  alert_at_50       BOOLEAN DEFAULT true,
  alert_at_80       BOOLEAN DEFAULT true,
  alert_at_100      BOOLEAN DEFAULT true,
  alert_50_sent     BOOLEAN DEFAULT false,  -- Évite doublons
  alert_80_sent     BOOLEAN DEFAULT false,
  alert_100_sent    BOOLEAN DEFAULT false,
  
  updated_at        TIMESTAMPTZ DEFAULT NOW()
);

-- Historique mensuel (pour reporting)
CREATE TABLE quota_history (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       UUID REFERENCES users(id),
  period        TEXT NOT NULL,             -- '2025-01' format
  hours_used    FLOAT NOT NULL,
  hours_allocated FLOAT,                   -- Snapshot à la fin du mois
  created_at    TIMESTAMPTZ DEFAULT NOW()
);

-- Fonction: incrémenter quota après transcription
CREATE OR REPLACE FUNCTION increment_user_quota(
  p_user_id UUID, 
  p_duration_hours FLOAT
) RETURNS VOID AS $$
DECLARE
  v_quota user_quotas%ROWTYPE;
  v_pct FLOAT;
BEGIN
  SELECT * INTO v_quota FROM user_quotas WHERE user_id = p_user_id FOR UPDATE;
  
  -- Incrémenter
  UPDATE user_quotas 
  SET hours_used_month = hours_used_month + p_duration_hours,
      hours_used_total = hours_used_total + p_duration_hours,
      updated_at = NOW()
  WHERE user_id = p_user_id;
  
  -- Calculer pourcentage
  IF v_quota.hours_allocated IS NOT NULL THEN
    v_pct := (v_quota.hours_used_month + p_duration_hours) / v_quota.hours_allocated * 100;
    
    -- Déclencher alertes
    IF v_pct >= 50 AND NOT v_quota.alert_50_sent AND v_quota.alert_at_50 THEN
      PERFORM pg_notify('quota_alert', json_build_object('user_id', p_user_id, 'pct', 50)::text);
      UPDATE user_quotas SET alert_50_sent = true WHERE user_id = p_user_id;
    END IF;
    IF v_pct >= 80 AND NOT v_quota.alert_80_sent AND v_quota.alert_at_80 THEN
      PERFORM pg_notify('quota_alert', json_build_object('user_id', p_user_id, 'pct', 80)::text);
      UPDATE user_quotas SET alert_80_sent = true WHERE user_id = p_user_id;
    END IF;
    IF v_pct >= 100 AND NOT v_quota.alert_100_sent AND v_quota.alert_at_100 THEN
      PERFORM pg_notify('quota_alert', json_build_object('user_id', p_user_id, 'pct', 100)::text);
      UPDATE user_quotas SET alert_100_sent = true WHERE user_id = p_user_id;
    END IF;
  END IF;
END;
$$ LANGUAGE plpgsql;

async def check_user_quota(user_id: str, duration_hours: float) -> QuotaCheck:
    quota = await db.get_quota(user_id)
    
    if quota.hours_allocated is None:
        return QuotaCheck(allowed=True, reason="unlimited")
    
    remaining = quota.hours_allocated - quota.hours_used_month
    
    if remaining <= 0:
        if quota.overage_policy == "block":
            return QuotaCheck(allowed=False, reason="quota_exceeded",
                               message="Quota mensuel épuisé. Contactez votre admin.")
        elif quota.overage_policy == "warn":
            return QuotaCheck(allowed=True, reason="overage_warn",
                               warning="Quota dépassé — traitement autorisé mais notifié")
        else:  # allow
            return QuotaCheck(allowed=True, reason="overage_allowed")
    
    return QuotaCheck(allowed=True, remaining_hours=remaining)

Politiques de dépassement

PolitiqueComportementCas d'usageNotification
block Le job est refusé avant de démarrer. Message d'erreur clair à l'utilisateur. Utilisateurs non-payants, essai limité, équipes avec budget strict Email user: "Quota atteint" + lien vers admin
warn Le job démarre normalement mais admin et user reçoivent une notification. Équipes de confiance, usage légèrement variable d'un mois à l'autre Email admin + user: "Quota dépassé — X heures en surplus"
allow Le job démarre sans notification. Consommation trackée pour reporting. Équipes internes, super admin, usage illimité avec suivi seul Aucune notification immédiate — visible dans reporting mensuel

  ROUND((uq.hours_used_month / NULLIF(uq.hours_allocated, 0) * 100)::numeric, 1) AS pct_used,
  uq.overage_policy,
  uq.reset_day
FROM users u
JOIN user_quotas uq ON uq.user_id = u.id
WHERE u.org_id = $1
ORDER BY pct_used DESC NULLS LAST;

-- Historique mensuel
SELECT period, SUM(hours_used) as total_org_hours
FROM quota_history
WHERE user_id IN (SELECT id FROM users WHERE org_id = $1)
GROUP BY period
ORDER BY period DESC
LIMIT 12;
